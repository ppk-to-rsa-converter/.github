## What's the Difference Between PPK and RSA (OpenSSH)?

Two containers, same car inside.

Your private key is just a big number — the cryptographic kind. The difference is in how it gets wrapped up and stored on disk.

**PPK** (PuTTY Private Key) is PuTTY's proprietary format. It bundles the public and private key together in one `.ppk` file. Great for PuTTY on Windows. Useless on Linux out of the box.

**RSA / OpenSSH format** is what Linux's `ssh` command expects. The private key lives in a file usually called `id_rsa` (or `id_ed25519`, or whatever you named it), starting with `-----BEGIN OPENSSH PRIVATE KEY-----` or `-----BEGIN RSA PRIVATE KEY-----`. This is the universal language of the SSH world.

When people say "convert PPK to RSA," they really mean: *export the private key from PuTTY's container format into a format OpenSSH can read*. No cryptography changes. Just a format translation.

## Step-by-Step: Convert PPK to OpenSSH (RSA) on Windows

This is the most common scenario — you're on Windows, you have a `.ppk` file from PuTTYgen or downloaded from DMIT, and you need to connect from WSL, Git Bash, or pass the key to a Linux/macOS machine.

### Method 1: Using PuTTYgen GUI (Simplest)

1. Open **PuTTYgen** (comes bundled with PuTTY, downloadable from the official PuTTY site)
2. Click **Load** — browse to your `.ppk` file. If you don't see it, change the file filter to `*.*`
3. Enter your passphrase if prompted
4. Go to **Conversions** in the menu bar
5. Choose **Export OpenSSH key** — this saves a PEM-format private key compatible with OpenSSH
6. Save the file, e.g. `mykey.pem` or `id_rsa` (no extension)
7. Set permissions: `chmod 600 mykey.pem` (critical — OpenSSH refuses to use keys that are too permissive)
8. Connect: `ssh -i ~/.ssh/mykey.pem root@your-vps-ip`

> **Note on key format versions:** PuTTYgen has two export options — "Export OpenSSH key" (produces `-----BEGIN OPENSSH PRIVATE KEY-----`) and "Export OpenSSH key (force new file format)". For most modern setups, either works. If you need legacy compatibility with very old servers, some tools expect `-----BEGIN RSA PRIVATE KEY-----` — in that case, use the older PEM export.

### Method 2: Command Line with puttygen (Linux/macOS)

If you're on Linux or macOS and already have `putty-tools` installed:

bash
# Install on Debian/Ubuntu
sudo apt-get install putty-tools

# Convert PPK to OpenSSH format
puttygen mykey.ppk -O private-openssh -o ~/.ssh/id_rsa

# Set correct permissions
chmod 600 ~/.ssh/id_rsa

# Connect to your VPS
ssh -i ~/.ssh/id_rsa root@your-vps-ip


If your `.ppk` is version 3 (generated by PuTTY 0.76 or newer) and the installed `puttygen` is too old, you might see: `puttygen: error loading 'key.ppk': PuTTY key format too new`. In that case, manually build a newer version from source or download PuTTY 0.76+ directly.

### Method 3: Copy Converted Key to Default SSH Identity

If you want passwordless, seamless SSH without specifying `-i` every time:

bash
puttygen mykey.ppk -O private-openssh -o /tmp/converted_key
cat /tmp/converted_key > ~/.ssh/id_rsa
chmod 600 ~/.ssh/id_rsa
rm /tmp/converted_key


Now `ssh root@your-vps-ip` works without any flags.

## Common Errors and What They Mean

| Error | Cause | Fix |
|---|---|---|
| `Permission denied (publickey)` | Wrong key, wrong user, or key not in `authorized_keys` | Verify key is uploaded to server; check permissions (`.ssh` = 700, `authorized_keys` = 600) |
| `WARNING: UNPROTECTED PRIVATE KEY FILE!` | Key file permissions too open | `chmod 600 ~/.ssh/id_rsa` |
| `PuTTY key format too new` | Old puttygen can't read PPK v3 | Upgrade puttygen or build 0.76+ from source |
| `no supported authentication methods` | Server only accepts key auth, no key provided | Pass key with `-i` flag or set default identity |
| `Enter passphrase` prompt on every login | Key is passphrase-protected but no agent running | Start `ssh-agent` and run `ssh-add ~/.ssh/id_rsa` |
